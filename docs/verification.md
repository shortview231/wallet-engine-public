# Verification

## What the public proof layer demonstrates

### Ledger overview

- current derived balance
- latest posted anchor balance
- oldest known balance context
- transaction load count
- missing posted-balance detection

### Ledger transactions

- normalized transaction rows
- amount and balance interpretation
- posted vs derived balance presentation
- filtering-friendly table structure

### Expense matching

- expected obligation registry
- paid and unpaid state handling
- last paid date and paid amount fields
- payment match counts

## Verification posture

The demonstration pack is intended to prove that the engine exists and has meaningful operational depth, while keeping all published values proxy-only.
